Create `src/lib/actions/player.ts` with a single exported server action `registerPlayer`.

The action signature:
```typescript
export async function registerPlayer(_prevState: unknown, formData: FormData)
```

It receives:
- `name` (string) from formData — the player's chosen display name
- `campaignId` (string) from formData — the campaign to join
- `joinToken` (string) from formData — needed to construct the redirect URL

Validation:
- Trim the name. If empty after trim, return `{ error: 'Please enter your name.' }`
- If name longer than 50 characters, return `{ error: 'Name must be 50 characters or fewer.' }`

On success:
1. Create a PlayerSlot: `prisma.playerSlot.create({ data: { campaignId, name: trimmedName } })`
2. Set httpOnly cookie `player_id` with value = `slot.id`, options: `{ httpOnly: true, path: '/', maxAge: 60 * 60 * 24 * 365, sameSite: 'lax' }` — same pattern as dm_secret. Must `await cookies()` before calling `.set()`.
3. `redirect(`/join/${joinToken}/availability`)` — Next.js redirect() throws (use outside try/catch or as final statement).

Import pattern: `'use server'` directive at top. Import `redirect` from `'next/navigation'`, `cookies` from `'next/headers'`, `prisma` from `'@/lib/prisma'`.

Do NOT use `secure: true` on the cookie (same decision as dm_secret — dev is HTTP, Vercel enforces HTTPS at platform level).
