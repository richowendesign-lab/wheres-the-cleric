  <files>src/app/page.tsx</files>
  <action>
    Convert `HomePage` to an async server component. At the top of the function body, before returning JSX:

    1. Read the `dm_secret` cookie using `cookies()` from `next/headers`.
    2. If the cookie exists, look up the campaign in the database using `prisma.campaign.findUnique({ where: { dmSecret: cookieValue }, select: { id: true } })`.
    3. If a matching campaign is found, call `redirect(`/campaigns/${campaign.id}`)` — this exits before any JSX renders.
    4. If the cookie is absent OR no matching campaign exists (e.g. DB was reset), fall through and render the existing home page JSX unchanged.

    Implementation pattern:
    ```typescript
    import { cookies } from 'next/headers'
    import { redirect } from 'next/navigation'
    import { prisma } from '@/lib/prisma'
    import Link from 'next/link'

    export default async function HomePage() {
      const cookieStore = await cookies()
      const dmSecret = cookieStore.get('dm_secret')?.value

      if (dmSecret) {
        const campaign = await prisma.campaign.findUnique({
          where: { dmSecret },
          select: { id: true },
        })
        if (campaign) {
          redirect(`/campaigns/${campaign.id}`)
        }
      }

      return (
        // existing JSX unchanged
      )
    }
    ```

    Do NOT throw if the campaign is not found — silently fall through so the DM can create a new campaign. This handles DB resets gracefully.
