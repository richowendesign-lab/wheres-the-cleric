<task type="auto">
  <name>Task 1: Configure Vercel deployment and deploy</name>
  <files>
    vercel.json
    .env.example
  </files>
  <action>
    Deploy the app to Vercel using the Vercel CLI. Vercel is the right choice here: zero-config for Next.js, free tier, stable URL, no server management.

    **Important note on SQLite + Vercel:** Vercel's filesystem is ephemeral (read-only in production). For Phase 1, use a persistent SQLite database via Vercel's file system workaround OR switch to a Vercel Postgres/Neon database for production. The simplest approach for a personal app with minimal traffic is to use Neon's free tier (PostgreSQL), which Vercel can provision automatically and has a Prisma adapter. However, if the user prefers keeping SQLite for simplicity, use a Turso SQLite edge database.

    **Decision: Use Neon PostgreSQL for production** (Vercel integrates it in one click, free tier, Prisma supports it natively). Local dev keeps SQLite for zero-config simplicity. Production uses Neon.

    Steps:

    Step 1 — Install Vercel CLI:
    ```
    npm install -g vercel
    ```

    Step 2 — Create vercel.json (minimal config for Next.js):
    ```json
    {
      "framework": "nextjs",
      "buildCommand": "prisma generate && next build",
      "installCommand": "npm install"
    }
    ```

    Step 3 — Update package.json to run `prisma generate` before build (Vercel runs build on each deploy):
    In package.json scripts, update or add:
    ```json
    "build": "prisma generate && next build",
    "postinstall": "prisma generate"
    ```

    Step 4 — Update prisma/schema.prisma to support both SQLite (local) and PostgreSQL (production) via environment variable switching. Replace the datasource block with:
    ```prisma
    datasource db {
      provider = env("DATABASE_PROVIDER")
      url      = env("DATABASE_URL")
    }
    ```

    Update .env (local) to add:
    ```
    DATABASE_PROVIDER="sqlite"
    DATABASE_URL="file:./dev.db"
    ```

    Update .env.example to document both:
    ```
    # Local development (SQLite - no setup needed)
    DATABASE_PROVIDER="sqlite"
    DATABASE_URL="file:./dev.db"

    # Production (set these in Vercel dashboard - Neon PostgreSQL)
    # DATABASE_PROVIDER="postgresql"
    # DATABASE_URL="postgresql://..."
    ```

    Step 5 — Deploy to Vercel:
    ```
    vercel --yes
    ```

    This will prompt for login if not already authenticated. After first deploy, run:
    ```
    vercel --prod
    ```

    Step 6 — In Vercel dashboard, set environment variables for production:
    - DATABASE_PROVIDER = postgresql
    - DATABASE_URL = (from Neon/Vercel Postgres integration)

    Note: If Vercel CLI authentication is needed, the executor will handle it as a dynamic auth gate.
