Create `src/app/join/[joinToken]/availability/page.tsx` as an async server component.

Page props: `params: Promise<{ joinToken: string }>` — await before destructuring (established project pattern).

Logic:
1. Await params: `const { joinToken } = await params`
2. Verify the campaign exists: `prisma.campaign.findUnique({ where: { joinToken } })`. If null, call `notFound()`.
3. Read `player_id` cookie: `cookieStore.get('player_id')?.value`. If absent, `redirect(`/join/${joinToken}`)` — sends them back to register.
4. Load the player slot with entries and campaign:
```typescript
const slot = await prisma.playerSlot.findUnique({
  where: { id: playerIdCookie },
  include: {
    availabilityEntries: true,
    campaign: {
      select: {
        id: true,
        joinToken: true,
        planningWindowStart: true,
        planningWindowEnd: true,
      },
    },
  },
})
```
5. If slot is null (stale cookie), `redirect(`/join/${joinToken}`)`.
6. If slot.campaignId does not match the campaign from step 2 (player cookie from a different campaign), `redirect(`/join/${joinToken}`)` — treats them as a new visitor.

Serialize data for AvailabilityForm:
```typescript
const serializedEntries = slot.availabilityEntries.map(e => ({
  id: e.id,
  type: e.type,
  dayOfWeek: e.dayOfWeek,
  date: e.date?.toISOString() ?? null,
  timeOfDay: e.timeOfDay,
  status: e.status,
}))
const windowStart = slot.campaign.planningWindowStart?.toISOString().split('T')[0] ?? ''
const windowEnd = slot.campaign.planningWindowEnd?.toISOString().split('T')[0] ?? ''
```
