  <name>Task 1: Rewrite schema.prisma to v1.1 model</name>
  <files>prisma/schema.prisma</files>
  <action>
    Replace the contents of prisma/schema.prisma with the v1.1 schema. Keep the existing generator and datasource blocks exactly as-is (Prisma 7 pattern — no url in datasource, PrismaNeon adapter is set at runtime).

    Changes to Campaign model:
    - REMOVE: name String
    - REMOVE: dmName String
    - ADD: joinToken String @unique @default(cuid())
    - ADD: dmSecret  String @unique @default(cuid())
    - KEEP: id, planningWindowStart, planningWindowEnd, createdAt, updatedAt, playerSlots relation

    Changes to PlayerSlot model:
    - REMOVE: inviteToken String @unique @default(cuid())
    - KEEP: id, campaignId, name, createdAt, updatedAt, campaign relation, availabilityEntries relation

    AvailabilityEntry model: no changes.

    Final schema.prisma should look like:

    ```prisma
    generator client {
      provider = "prisma-client"
      output   = "../src/generated/prisma"
    }

    datasource db {
      provider = "postgresql"
    }

    model Campaign {
      id                  String       @id @default(cuid())
      joinToken           String       @unique @default(cuid())
      dmSecret            String       @unique @default(cuid())
      planningWindowStart DateTime?
      planningWindowEnd   DateTime?
      createdAt           DateTime     @default(now())
      updatedAt           DateTime     @updatedAt
      playerSlots         PlayerSlot[]
    }

    model PlayerSlot {
      id                  String              @id @default(cuid())
      campaignId          String
      name                String
      createdAt           DateTime            @default(now())
      updatedAt           DateTime            @updatedAt
      campaign            Campaign            @relation(fields: [campaignId], references: [id], onDelete: Cascade)
      availabilityEntries AvailabilityEntry[]
    }

    // type: "weekly" = recurring day-of-week pattern
    // type: "override" = specific date exception
    model AvailabilityEntry {
      id           String     @id @default(cuid())
      playerSlotId String
      type         String     // "weekly" | "override"
      dayOfWeek    Int?       // 0=Sun, 1=Mon, ..., 6=Sat (used when type="weekly")
      date         DateTime?  // specific date (used when type="override")
      timeOfDay    String?    // "morning" | "afternoon" | "evening"
      status       String     // "free" | "busy"
      createdAt    DateTime   @default(now())
      updatedAt    DateTime   @updatedAt
      playerSlot   PlayerSlot @relation(fields: [playerSlotId], references: [id], onDelete: Cascade)

      @@unique([playerSlotId, date])
    }
    ```
