```typescript
// The click handler pattern:
function handleDateClick(dateKey: string) {
  const isCurrentlyBlocked = exceptions.has(dateKey)
  const prevExceptions = exceptions                    // save for rollback
  const newExceptions = new Set(exceptions)
  if (isCurrentlyBlocked) newExceptions.delete(dateKey)
  else newExceptions.add(dateKey)

  setExceptions(newExceptions)   // optimistic update
  setSaveStatus('saving')

  toggleDmException(campaignId, dateKey, !isCurrentlyBlocked)
    .then(result => {
      if ('error' in result) {
        setSaveStatus('error')
        setExceptions(prevExceptions)  // rollback
      } else {
        setSaveStatus('saved')
        setTimeout(() => setSaveStatus('idle'), 2000)
      }
    })
    .catch(() => {
      setSaveStatus('error')
      setExceptions(prevExceptions)  // rollback
    })
}
```

    Create `src/components/DmExceptionCalendar.tsx` as a new `'use client'` component. This is a self-contained click-to-toggle calendar for DM-unavailable dates with optimistic state and a block/flag mode toggle button.

    **Props interface:**
    ```typescript
    interface DmExceptionCalendarProps {
      campaignId: string
      planningWindowStart: string  // 'YYYY-MM-DD'
      planningWindowEnd: string    // 'YYYY-MM-DD'
      initialExceptions: string[]  // 'YYYY-MM-DD' array
      exceptionMode: 'block' | 'flag' | null
    }
    ```

    **State:**
    - `exceptions: Set<string>` — initialised from `new Set(initialExceptions)`
    - `saveStatus: SaveStatus` — from `@/components/Toast`
    - `mode: 'block' | 'flag'` — initialised from `exceptionMode ?? 'block'`
    - `modeStatus: 'idle' | 'saving' | 'saved' | 'error'` — separate status for mode toggle

    **Imports:**
    - `'use client'`
    - `useState` from react
    - `toggleDmException, setDmExceptionMode` from `@/lib/actions/campaign`
    - `buildMonthGrid, formatDateKey` from `@/lib/calendarUtils`
    - `Toast, SaveStatus` from `@/components/Toast`

    **handleDateClick:** Follow the optimistic pattern exactly as shown in the interfaces block. The toggle logic: if `exceptions.has(dateKey)` → set `isBlocked = false` (removing); else → set `isBlocked = true` (adding).

    **handleModeToggle:** Switches between 'block' and 'flag'. Optimistically update local `mode` state, call `setDmExceptionMode(campaignId, newMode)`. On error, roll back mode to previous value and set `modeStatus('error')`. On success, set `modeStatus('saved')`, clear after 2s.

    **Calendar grid rendering:** Build months list from `planningWindowStart` to `planningWindowEnd` (same pattern as AvailabilityCalendar — iterate by month, use `getUTCFullYear`/`getUTCMonth` to avoid timezone shifts). For each month, use `buildMonthGrid(year, month)`.

    **Cell state logic:**
    ```typescript
    type CellState = 'outside-window' | 'dm-blocked' | 'normal'

    function getCellState(date: Date, windowStart: Date, windowEnd: Date, exceptions: Set<string>): CellState {
      const dateUTC = new Date(Date.UTC(date.getUTCFullYear(), date.getUTCMonth(), date.getUTCDate()))
      if (dateUTC < windowStart || dateUTC > windowEnd) return 'outside-window'
      const dateKey = formatDateKey(date)
      return exceptions.has(dateKey) ? 'dm-blocked' : 'normal'
    }
    ```

    **Cell styling:**
    - `outside-window`: `text-gray-700 cursor-default` (non-interactive)
    - `dm-blocked`: `bg-amber-500/20 text-amber-400 ring-1 ring-amber-400/50 cursor-pointer hover:opacity-70 transition-opacity rounded-md`
    - `normal`: `text-gray-500 cursor-pointer hover:bg-amber-500/10 hover:text-amber-400 transition-colors rounded-md`

    Amber/orange is the DM-blocked colour token — visually distinct from player availability (green = free, gray = no-response).

    **Mode toggle button:** Render above or below the calendar grid. Shows current mode as a pill/toggle:
    ```tsx
    <div className="flex items-center gap-3 mb-3">
      <span className="text-sm text-gray-400">Blocked dates:</span>
      <button
        type="button"
        onClick={handleModeToggle}
        className={`text-xs px-3 py-1 rounded-full border transition-colors ${
          mode === 'block'
            ? 'bg-red-900/40 border-red-700/50 text-red-300'
            : 'bg-amber-900/30 border-amber-700/40 text-amber-300'
        }`}
      >
        {mode === 'block' ? 'Block (hide from rankings)' : 'Flag (warn in rankings)'}
      </button>
      {modeStatus === 'saving' && <span className="text-xs text-gray-500">Saving...</span>}
    </div>
    ```

    **Legend:** Add a legend below the calendar grid:
    ```tsx
    <div className="flex flex-wrap gap-3 text-xs text-gray-500 mt-3">
      <span className="flex items-center gap-1.5">
        <span className="inline-block w-3 h-3 rounded bg-amber-500/20 ring-1 ring-amber-400/50" />
        DM unavailable
      </span>
      <span className="flex items-center gap-1.5">
        <span className="inline-block w-3 h-3 rounded bg-gray-800" />
        Available
      </span>
    </div>
    ```

    **Toast:** Render `<Toast status={saveStatus} onRetry={() => setSaveStatus('idle')} onDismiss={() => setSaveStatus('idle')} />` — the retry for DM exceptions is a no-op (just resets to idle; user can click again). No debounce needed — each click is a single atomic action.

    **Section header:** Wrap in a `<section>` with heading: `<h2 className="text-lg font-semibold text-white mb-3">My Unavailable Dates</h2>`

    The component does NOT call revalidatePath after toggleDmException — optimistic state is the source of truth for exception toggle. It DOES get re-mounted when setDmExceptionMode triggers revalidatePath (Server Component re-renders, new props flow down).
