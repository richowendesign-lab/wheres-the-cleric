Create `src/components/DatePickerInput.tsx` as a `'use client'` component. This is a self-contained date picker that wraps `buildMonthGrid` and `formatDateKey` from `@/lib/calendarUtils` — do NOT import any date library.

**Props interface:**
```typescript
interface DatePickerInputProps {
  name: string          // FormData field name, e.g. "planningWindowStart"
  defaultValue?: string // 'YYYY-MM-DD' string — used when editing existing value
  required?: boolean
  placeholder?: string  // Default: 'Pick a date'
}
```

**State:**
- `open: boolean` — controls popover visibility
- `selected: Date | undefined` — the picked date; initialized from `defaultValue` via `parseDateKey`
- `viewYear: number` — the month the calendar is showing
- `viewMonth: number` — 0-indexed

**parseDateKey helper (module-level, NOT exported):**
```typescript
function parseDateKey(dateKey: string): Date {
  const [y, m, d] = dateKey.split('-').map(Number)
  return new Date(Date.UTC(y, m - 1, d))
}
```

**Outside-click dismiss:** Use `containerRef` (`useRef<HTMLDivElement>`). Listen for both `mousedown` AND `touchstart` (for mobile) when `open` is true. Check `containerRef.current.contains(e.target as Node)` — close if outside. Clean up listener on effect cleanup.

**Escape-key dismiss:** Separate `useEffect` listening for `keydown` when `open` is true.

**Month navigation:**
```typescript
const MONTHS = ['January','February','March','April','May','June','July','August','September','October','November','December']
const DAY_HEADERS = ['Su', 'Mo', 'Tu', 'We', 'Th', 'Fr', 'Sa']

function prevMonth() {
  if (viewMonth === 0) { setViewMonth(11); setViewYear(y => y - 1) }
  else setViewMonth(m => m - 1)
}
function nextMonth() {
  if (viewMonth === 11) { setViewMonth(0); setViewYear(y => y + 1) }
  else setViewMonth(m => m + 1)
}
```

**Hidden input:** Place AFTER the trigger button within the component's `<div>`. Use `<input type="hidden" name={name} value={formattedValue} />` where `formattedValue = selected ? formatDateKey(selected) : ''`. Do NOT add `required` to the hidden input — browsers ignore `required` on hidden inputs; server-side validation already handles missing dates.

**Trigger button:** `type="button"` (critical — prevents form submission on click). Show `displayLabel` = `selected.toLocaleDateString('en-GB', { day: 'numeric', month: 'short', year: 'numeric', timeZone: 'UTC' })` when selected, else `placeholder`. Classes mirror existing `inputCls`: `w-full text-left rounded bg-[var(--dnd-input-bg)] border border-[var(--dnd-accent)] px-3 py-2 text-gray-100 focus:outline-none`.

**Popover:** `position: absolute` via `className="absolute z-50 mt-1 ..."`, parent container has `position: relative`. Background `bg-[#140326]`, border `border border-[var(--dnd-border-card)]`, `shadow-2xl`, `rounded-lg`, `p-3`, `min-w-[260px]`.

**Calendar grid rendering:**
- Day header row: 7-column grid, each cell `text-xs text-gray-500 text-center py-1`
- Weeks: iterate `grid = buildMonthGrid(viewYear, viewMonth)`. For null cells render `<div key={di} />`. For date cells:
  - `isSelected = selected && formatDateKey(selected) === formatDateKey(date)`
  - Selected: `bg-[var(--dnd-accent)] text-black font-semibold`
  - Unselected: `text-gray-300 hover:bg-[var(--dnd-border-card)] hover:text-white`
  - Display: `date.getUTCDate()` (NOT `getDate()` — timezone safety)
  - All day buttons: `type="button"`, `rounded`, `py-1.5`, `text-sm`, `text-center`, `transition-colors`

**Month nav buttons:** `type="button"`, prev `aria-label="Previous month"` with `←` (HTML entity `&#8592;`), next `aria-label="Next month"` with `→` (`&#8594;`). Classes: `p-1 text-[var(--dnd-text-muted)] hover:text-white`.

Export as named export: `export function DatePickerInput(...)`.
