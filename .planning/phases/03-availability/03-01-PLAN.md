    **Action 1: `saveWeeklyPattern`**
    - Signature: `export async function saveWeeklyPattern(playerSlotId: string, entries: { dayOfWeek: number; timeOfDay: string }[]): Promise<{ success: true } | { error: string }>`
    - Uses Prisma `$transaction` to atomically delete all `type: 'weekly'` entries for the playerSlotId, then recreate them with `createMany` (or multiple `create` calls inside the transaction if `createMany` causes issues with the SQLite adapter — test `createMany` first).
    - Each created entry: `{ playerSlotId, type: 'weekly', dayOfWeek: e.dayOfWeek, timeOfDay: e.timeOfDay, status: 'free' }`
    - Do NOT call `revalidatePath` — the client manages its own state (per research pitfall #3: revalidatePath causes form state loss and flicker).
    - On error: catch + `console.error` + return `{ error: 'Failed to save. Please try again.' }`

    **Action 2: `toggleDateOverride`**
    - Signature: `export async function toggleDateOverride(playerSlotId: string, date: string, status: 'free' | 'busy' | null): Promise<{ success: true } | { error: string }>`
    - `date` is an ISO date string ("YYYY-MM-DD"). Convert to `Date` using `new Date(Date.UTC(year, month, day))` to avoid timezone shift (parse year/month/day from the string: `const [y, m, d] = date.split('-').map(Number)`).
    - If `status === null`: delete the override entry (`deleteMany` where `{ playerSlotId, type: 'override', date: parsedDate }`).
    - If `status === 'free' | 'busy'`: upsert using `where: { playerSlotId_date: { playerSlotId, date: parsedDate } }` (the composite accessor generated from `@@unique([playerSlotId, date])`). Create fields: `{ playerSlotId, type: 'override', date: parsedDate, status }`.
    - Do NOT call `revalidatePath`.
    - On error: catch + `console.error` + return `{ error: 'Failed to save. Please try again.' }`
  </action>
