    Edit prisma/schema.prisma with two additions:

    1. In the Campaign model, add two lines after the existing `playerSlots PlayerSlot[]` relation:
       ```
       dmExceptionMode          String?
       dmAvailabilityExceptions DmAvailabilityException[]
       ```

    2. After the Session model (at the end of the file), add the new model:
       ```prisma
       model DmAvailabilityException {
         id         String   @id @default(cuid())
         campaignId String
         date       DateTime
         createdAt  DateTime @default(now())
         campaign   Campaign @relation(fields: [campaignId], references: [id], onDelete: Cascade)

         @@unique([campaignId, date])
       }
       ```

    Implementation notes:
    - date is DateTime (not DateTime?) — every exception record must have a date; nullable is wrong here
    - No updatedAt field — exceptions are toggled (delete + create), not updated in place
    - onDelete: Cascade — DM exception records are meaningless without their campaign; this mirrors PlayerSlot and AvailabilityEntry cascade behaviour
    - @@unique([campaignId, date]) mirrors AvailabilityEntry's @@unique([playerSlotId, date])
    - dmExceptionMode String? on Campaign holds "block" or "flag" as plain strings; nullable means "not yet set" (default to block behaviour in application logic)
    - Do NOT add a @@map, @@index, or any constraint beyond @@unique — keep it simple, matching the AvailabilityEntry pattern
  </action>
